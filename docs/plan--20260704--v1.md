## M0 — 骨架与规范

**目标**：可运行的空骨架，四层分层就位，配置集中。

- [ ] `pyproject.toml` / 依赖锁定（langgraph、llama-index、chromadb、sentence-transformers、sqlite、streamlit、ragas 等）
- [ ] 目录结构（见 design §10）
- [ ] `config.yaml`：模型/向量库后端/路径/开关
- [ ] 模型层接口 + DeepSeek 接入 + 本地 bge-m3 加载
- [ ] 存储层 `VectorStore` 接口 + Chroma 实现（Milvus 留桩）
- [ ] SQLite schema（papers/notes/citations）
- [ ] 一个"hello"CLI 命令跑通全链路空调用
- **Demo**：`georesearcher --version` + config 切换后端生效。

## M1 — RAG 内核重构（融合"研问"）✅

**目标**：把旧项目升级为分层、可测的 RAG 内核。**（已完成 2026-07-04）**

- [x] ingest：PyMuPDF 解析 + 父子分块 + 批量导入（93 篇论文, 25,827 段落, 629 section 父块）
- [x] rag：HyDE + 多查询并行 + **混合检索（稠密 bge-m3 + 稀疏 BM25 rank_bm25）** + **RRF 融合（k=60）** + **交叉编码器重排序（bge-reranker-v2-m3）** + TTLCache 缓存
- [x] 生成：强制引用 grounding + APA 格式
- [x] CLI：`georesearcher ask "问题"` 完整问答
- [x] 批量向量化工具：`scripts/batch_ingest.py`（断点续跑 + tqdm 进度条 + status 命令）
- [ ] 吸收 PaperQA2：metadata 过滤、（可选）Unpaywall 找 OA（待 M3）
- **Demo**：投喂 PDF → 提问 → 带 APA 引用的答案。

## M2 — RAG 评估（做深①，面试最高价值）⭐

**目标**：内建可复现的 RAG 评估闭环。

- [ ] 评估集构建：LLM 半自动造 QA + 人工校验（存 tests/eval_set）
- [ ] 检索层指标：Hit Rate/Recall@k、MRR、NDCG、Context Precision/Recall
- [ ] 生成层指标：RAGAS（Faithfulness/Answer Relevancy）+ judge model（可配置）
- [ ] LLM-as-judge 偏差处理（不同 judge 交叉、人工抽检）
- [ ] 评估报告输出（表格 + 定位建议）
- [ ] 回归脚本：改动前后跑同一评估集对比
- **Demo**：`georesearcher eval` → 生成一份分层评估报告，能指出"检索层弱还是生成层弱"。
- **面试锚点**：这一步做完，"如何评测 RAG"你就能拿实物 + 话术回答。

## 风险与对策

| 风险 | 对策 |
|---|---|
| 贪全导致烂尾 | 严守"做深3+demo2+占位"，M2/M4/M5 是必达，其余可砍 |
| AI 写代码越写越乱 | M0 先立分层与测试硬约束（ADR-07） |
| QGIS MCP 环境难配 | gis-mcp（纯库、无需 QGIS）作为主力，qgis_mcp 作为加分 demo |
| 评估集造得不严谨 | 人工校验 + judge 交叉，宁少而准 |
| Milvus 破坏可跑性 | 默认 Chroma，Milvus 仅作可选（ADR-03） |

## 建议节奏

优先级：**M0 → M1 → M2（面试价值最高，尽早做完）→ M4 → M5 → M3 → M6 → M7**。
即使只做到 M2，你已经能在简历/面试里讲"可评估的 RAG 系统"；做到 M6 就是完整闭环 demo。

## 0. 本轮 grill 敲定的决策（TL;DR）

| # | 决策 | 选择 | 备选 | 理由 / trade-off |
|---|---|---|---|---|
| **范围** | M3 装多大 | **只做「检索入库最小闭环」** | 把编排/MCP 也塞进来 | 作品集真正重心是后续的 **LangGraph 编排 + MCP**（体现对 Agent 的理解）；文献库仍以**手动 PDF 为主**。检索入库只求"跑通流程、面试能说"，不追全。编排、MCP 各自作为**后续独立里程碑**。 |
| **A1** | 检索源 | **只打通 OpenAlex** + 抽象 `PaperSource` 接口，arXiv/S2 留桩 | 三源全接 | OpenAlex 免费无 key、一个请求同时给元数据+OA 链接、覆盖社科最广；arXiv 偏 CS/物理命中低；S2 无 key 限流严。窄而透（ADR-10）。 |
| **A2** | 合规/限流 & 无全文处理 | polite `mailto`+客户端限速；**仅下 OA 全文**；无全文走**方案(a)**：元数据+摘要入库，标 `oa_status` | (b) 无全文整篇跳过 | 只从 OpenAlex 返回的 OA 链接下载，禁猜 URL/盗版源（ADR-09）；摘要对解读归档/引用关系仍有价值。 |
| **B1** | 去重键 & 覆盖策略 | 统一 `paper_id` = **DOI → OpenAlex id → title 哈希**；下载前判重；命中已有论文用 OpenAlex 元数据**覆盖更新但不重向量化** | 沿用 PDF 内容哈希 | PDF 哈希导致"元数据阶段 id ≠ PDF 解析 id"、无法下载前判重；DOI 稳定且下载前可得。 |
| **B2** | 摘要-only 入库旁路 | 新增 `ingest_metadata_only()`；摘要作 `level="abstract"` 单块入向量库；OpenAlex inverted-index 摘要**在 search 层重建成纯文本** | 摘要不入向量库 | A2(a) 要求摘要可被 RAG 检索到；ingest 层只认纯文本、不关心来源格式（职责分离）。 |
| **C1** | interpret 形态 & 可靠性 | 固定 JSON schema 抽 5 字段；**缺失留空、禁编造**；基于全文或摘要（如实标注）；做成**独立命令 `interpret <paper_id>` 按需触发** | ingest 时自动跑 | 逐篇调 LLM 慢又费钱，按需更可控、好演示；缺失留空延续 M2 的"可信度"纪律。 |
| **E1** | 交付 & 隔离 | CLI 三命令；外部 API/PDF/LLM **全 mock 测试**；**M3 用独立库路径隔离 M2 评估库** | 与 M2 共用一个库 | 保证 M2 基线（36 条 eval_set，93 篇快照）可复现，不被新灌论文污染检索池。 |

## 1. M3 目标与非目标

### 1.1 目标（做到即 M3 完成）
输入一个研究主题 → 从 OpenAlex 检索候选文献 → 判重后统一入库（有 OA 全文走完整解析，无全文只存元数据+摘要）→ 逐篇按需生成结构化笔记（RQ/method/contribution/gap/…）落 SQLite。

**Demo（对齐 plan §M3）**：
```
georesearcher search "urban vitality POI" --limit 10       # 看候选
georesearcher ingest-search "urban vitality POI" --limit 10 # 检索+判重+入库
georesearcher interpret <paper_id>                          # 生成结构化笔记
```

### 1.2 非目标（M3 明确不做，避免范围蔓延）
- ❌ 不接 arXiv / Semantic Scholar 实际调用（只留 `PaperSource` 接口桩）。
- ❌ 不做 LangGraph 编排（下一个独立里程碑）。
- ❌ 不接任何 MCP（gis-mcp/qgis_mcp，独立里程碑）。
- ❌ 不做复杂的多源去重合并、引用关系自动抽取（`citations` 表本轮不写）。
- ❌ 不追求 OpenAlex 检索的召回质量/排序调优——**跑通即可**。

**分层红线**：search 层负责"和外部世界打交道"（HTTP、格式重建），产出干净的 `Paper` + 纯文本摘要/PDF 路径；ingest 层只认本地 PDF 路径或纯文本，不知道 OpenAlex 存在；interpret 层只认 `paper_id` + 从 store 读到的上下文，不碰网络。三层通过 `Paper`/`Chunk`/`StructuredNote`（`types.py`）+ store 接口通信。

### 3.4 `capabilities/search/pipeline.py` — 检索→判重→入库编排（不含 LangGraph，纯函数编排）

```python
class IngestSearchResult(BaseModel):
    query: str
    total_hits: int
    ingested_full: list[str]      # 走全文 ingest 的 paper_id
    ingested_meta: list[str]      # 走摘要旁路的 paper_id
    skipped_existing: list[str]   # 判重命中、跳过下载的 paper_id
    updated_meta: list[str]       # 命中已有、仅覆盖更新元数据的 paper_id
    failed: list[dict]            # {paper_id, reason}


def ingest_from_search(
    query: str, *, limit: int = 10,
    source: PaperSource | None = None,        # 默认 OpenAlexSource
    cfg: Config | None = None,
    sqlite_store: SqliteStore | None = None,
    vector_store: VectorStore | None = None,
    embedder: Embedder | None = None,
    downloader=None,                          # 可注入 fake（测试用）
) -> IngestSearchResult:
    """
    for hit in source.search(query, limit=limit):
      existing = sqlite_store.get_paper(hit.paper.id)
      if existing:                       # B1 判重
          sqlite_store.add_paper(merge_meta(existing, hit.paper))  # 覆盖更新，不重向量化
          → updated_meta; continue
      if hit.pdf_url:                    # 有 OA 全文
          path = downloader.download(hit.pdf_url)   # 失败→failed，不中断
          if path: ingest_paper(path, paper_id=hit.paper.id, ...) → ingested_full
      else:                              # A2(a) 无全文
          ingest_metadata_only(hit.paper, hit.abstract, ...) → ingested_meta
    """
```

## 7. ADR 追加（写进 design §11）

**ADR-12 M3 单源检索 + 接口留桩**：选 **OpenAlex 单源 + `PaperSource` 接口抽象，arXiv/S2 留桩**。备选：三源全接。理由：OpenAlex 免费无 key、一请求给元数据+OA 链接、覆盖社科最广；作品集重心是编排/MCP，检索入库"跑通即可"（窄而透，ADR-10）。Trade-off：召回覆盖不如多源，但接口已留好扩展点。

**ADR-13 统一去重键 DOI 优先**：`paper_id` = DOI(规范化) → OpenAlex id → title 哈希；下载前判重；命中已有仅覆盖元数据不重向量化。备选：沿用 PDF 内容哈希。理由：PDF 哈希导致元数据阶段与解析阶段 id 不一致、无法下载前判重、浪费算力。Trade-off：`ingest_paper` 需加可选 `paper_id` 参数。

**ADR-14 无 OA 全文→元数据+摘要入库**：无全文的论文走 `ingest_metadata_only`，摘要作 `level="abstract"` 单块入向量库、标 `oa_status="metadata_only"`。备选：整篇跳过。理由：摘要对解读/检索/引用仍有价值；合规红线只从 OA 链接下载（ADR-09）。Trade-off：库里混入"仅摘要"论文，检索需按 `oa_status` 区分。

**ADR-15 M3 独立库路径隔离 M2 基线**：M3 demo 用 `config.m3.yaml` 指向 `./data/m3/`。备选：共用一个库。理由：保证 M2 的 36 条 eval_set 基线（93 篇快照）可复现，不被新灌论文污染检索池。Trade-off：两份库、demo 时要记得指对 config。

## 11. 端到端验证记录（2026-07-05）

### 11.1 验证目标

在真实网络环境下跑通「OpenAlex 检索 → 判重 → OA PDF 下载 → ingest 入库 → interpret 结构化解读」完整闭环。

### 11.2 结论

**端到端验证未完成**——OpenAlex API 在真实调用时遇到网络层阻断，mock 测试全部通过（174/174 green），但真实 HTTP 请求未能成功建立。

### 11.3 问题诊断过程（完整排障链，面试素材）

| 步骤 | 操作 | 结果 | 排查方向 |
|---|---|---|---|
| 1 | `uv run georesearcher search ... --config config.m3.yaml` | SSL handshake timeout（`_ssl.c:999`） | Python urllib 与公司网络不兼容 |
| 2 | `curl https://api.openalex.org/works?...` | **200 OK**，能拿到 JSON | 网络可达，问题在 Python SSL 层 |
| 3 | 给 OpenAlexSource 加不验证 SSL context | 仍然 timeout | 不是证书链问题 |
| 4 | 改 `urllib.request.urlopen` → 手动 `socket.create_connection` + SSL | 仍然 timeout | 不是 urllib 的 bug |
| 5 | 细化：分步测 TCP connect → SSL wrap_socket | **TCP connect OK（2.2s）**，SSL wrap_socket **timeout** | TCP 层面能握手，TLS 层面被搁置 |
| 6 | curl 在同一时刻仍能拿到 200 | curl TLS 指纹 OK，Python TLS 指纹被拦截 | **根因：Cloudflare 的 TLS fingerprinting 防护** |

### 11.4 根因分析

**Cloudflare TLS fingerprinting**：OpenAlex 前面有 Cloudflare CDN。curl 使用系统原生 TLS 库（SecureTransport），Python 3.11 使用 OpenSSL。两者的 TLS 握手指纹不同，Cloudflare 将 Python/OpenSSL 的 TLS 指纹识别为自动化流量，在 TLS 握手阶段直接搁置连接（既不接受也不拒绝，直到客户端超时）——这是 Cloudflare 针对 bot 的典型 DDoS 缓解行为。

此外，早期调试时 curl 连续快速请求触发了 OpenAlex 的 429 → 503 限流升级，导致 IP 被临时加入更严格的防护策略，加剧了 TLS 层面的阻断。

### 11.5 影响范围

- **M3 单元测试**：不受影响（全 mock，20 个 OpenAlex 测试全绿）
- **M3 集成测试**：不受影响（`ingest_from_search` + `ingest_metadata_only` 测试全 mock）
- **M3 CLI demo**：`search` 和 `ingest-search` 命令在真实网络环境下不可用
- **手动 ingest + interpret**：不受影响——用户仍可通过 `uv run georesearcher ingest <pdf> --config config.m3.yaml` 手动投喂 PDF，再 `uv run georesearcher interpret <paper_id>` 生成结构化笔记，完整闭环仍可演示
- **M2 基线**：完全不受影响（M3 用 `config.m3.yaml` → `./data/m3/` 独立库路径）

### 11.6 后续可尝试的解决方案（面试时可展示的"想过"）

1. **等 Cloudflare 封禁自动过期**（通常几小时到 24 小时），之后降低请求频率（加更长的客户端限速间隔）
2. **在公司外网环境运行**（如家用网络，无企业防火墙/代理干扰）
3. **走 HTTPS 代理**（如果公司有 HTTP 代理，`urllib` 可配 `ProxyHandler`）
4. **用 `curl` 作为传输层**（`subprocess` 调 `curl` 获取 JSON，Python 只负责解析——应急方案，不适合长期使用）
5. **用 `httpx` 替换 `urllib`**（`httpx` 使用不同的 HTTP 库底层，TLS 指纹可能不同；但需确认 pyproject 是否已有依赖）

### 11.7 面试话术建议

"我打通了 OpenAlex 的检索→入库完整链路，mock 测试全覆盖。实际跑的时候遇到 Cloudflare 的 TLS fingerprinting 防护——curl 能通但 Python urllib 的 TLS 指纹被识别为 bot。我做了完整的分层排障：从 SSL context → socket 层面分步测 TCP/TLS，最终定位到 Cloudflare CDN 层。这个排障过程本身就能体现我对网络栈的理解。而且说实话——这块开源数据本来就不全，我以手动投喂的 PDF 为主，检索入库只是打通闭环的一环。这个'诚实边界'反而比硬吹'全自动'更可信。"
